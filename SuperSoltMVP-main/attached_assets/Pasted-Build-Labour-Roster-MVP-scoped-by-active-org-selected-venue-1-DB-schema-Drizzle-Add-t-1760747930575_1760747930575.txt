Build Labour → Roster MVP scoped by active org + selected venue.

1) DB schema (Drizzle):
- Add tables:
  Staff (id, orgId FK, venueId FK nullable, name, email, roleTitle, hourlyRateCents, createdAt)
  Roster (id, orgId FK, venueId FK, weekStartDate DATE, createdAt, UNIQUE(orgId, venueId, weekStartDate))
  Shift (id, rosterId FK, staffId FK, roleTitle, startTs TIMESTAMPTZ, endTs TIMESTAMPTZ, breakMinutes INT default 0, createdAt)
  Add indexes: Staff(orgId), Roster(orgId, venueId), Shift(rosterId)
  All timestamps UTC.

2) Seed:
- Insert 3 Staff for Demo Bistro CBD: { name: 'Carla', roleTitle: 'FOH' }, { name: 'Mick', roleTitle: 'Bar' }, { name: 'Jo', roleTitle: 'Kitchen' } with hourlyRateCents 2800.
- Create a Roster for the current week for Demo Bistro CBD with 2 example Shifts.

3) API routes (App Router):
- GET /api/labour/roster?weekStart=YYYY-MM-DD
   • Read cookie "orgId". Require it.
   • Determine venue from localStorage 'venueId' header fallback: allow query ?venueId=...
   • Return { roster, staff, shifts } for that org+venue+week (create empty roster if none).
- POST /api/labour/shifts
   • Body: { rosterId, staffId, roleTitle, startTs, endTs, breakMinutes? }
   • requireRole(orgId, ['owner','manager'])
   • Insert Shift, wrap with withAudit('shift.create', before=null, after=shift)
   • Return created shift.

4) UI:
- Create page app/labour/roster/page.tsx
   • Weekly grid (Mon–Sun, 06:00–24:00) minimal: list shifts as blocks under each day.
   • Toolbar: date range picker (week), venue selector reuses current <OrgVenueSwitcher /> state (read-only), and “Add shift” button.
   • “Add shift” opens a dialog with Staff select, roleTitle, start/end datetime, break minutes. On save -> POST /api/labour/shifts then refresh roster.

5) Rules:
- All queries/mutations must assert membership via requireOrg(orgId) and be filtered by orgId (and venueId where relevant).
- TypeScript strict, no any. Keep styling simple with shadcn/ui.
