Goal: Link menu items ↔ recipes ↔ ingredients (including nested items), add supplier pricing, and enable sales import (manual + CSV now, OCR stub). Keep currency AUD (en-AU). Don’t touch invites/auth.

1) Database (Drizzle) – add/adjust tables

Create/modify schema files and push migration.

suppliers

id (pk), orgId (fk), name (text, required), contactEmail (text, nullable), phone (text, nullable), notes (text, nullable), isActive (boolean, default true), createdAt/updatedAt.

ingredient_suppliers (many suppliers per ingredient w/ prices)

id (pk), orgId (fk), ingredientId (fk), supplierId (fk), packSize (numeric, nullable), packUnit (text, nullable), unitPriceCents (int, required), leadTimeDays (int, default 0), sku (text, nullable), isPreferred (boolean, default false), unique(orgId, ingredientId, supplierId).

recipes (header)

id (pk), orgId (fk), menuItemId (fk), yieldQty (numeric, default 1), yieldUnit (text, nullable), wastagePct (numeric, default 0), unique(orgId, menuItemId).

recipe_lines (lines can reference an ingredient OR another menu item to allow nested recipes)

id (pk), orgId (fk), recipeId (fk), ingredientId (fk, nullable), subMenuItemId (fk, nullable), qty (numeric, required), unit (text, required), notes (text, nullable).

CHECK constraint: exactly one of (ingredientId, subMenuItemId) is not null.

daily_sales (keep existing) – no change, but add index on (orgId, date).

menu_items – add optional fields now (don’t break UI): sku (text, nullable), category (text, nullable), isComposite (computed by presence of recipe; store as boolean default false if easier).

2) Server utilities

In lib/currency.ts, keep AUD formatting helpers as source of truth. Ensure all new UI uses them.

Add lib/uom.ts with a small helper to normalize unit strings and convert g↔kg, ml↔L for cost/unit math (simple conversion map; we only need g/kg and ml/L for now).

3) API routes

Create REST endpoints with zod validation and orgId from cookie. Use existing patterns.

app/api/suppliers (GET list, POST create)

app/api/suppliers/[id] (GET, PATCH, DELETE – soft delete with isActive=false)

app/api/ingredient-suppliers (POST create, PATCH/DELETE on [id])

Recipes:

app/api/recipes (GET by menuItemId, POST create or upsert)

app/api/recipes/[id] (GET, PATCH, DELETE)

app/api/recipe-lines (POST add line, PATCH/DELETE on [id])

Sales import:

app/api/sales/import/csv (POST: accepts CSV text or file upload, returns parsed rows & creates daily_sales rows)

app/api/sales/import/ocr (POST: stub now – upload image, run Tesseract if available; otherwise return 501 with shape the UI expects)

Business rules in APIs

Costing: expose a helper getIngredientUnitCost(ingredientId) = cheapest active supplier unitPriceCents (or null).

Recipe costing: getMenuItemCost(menuItemId) recursively sums lines; handles:

ingredient lines → qty × normalized unit cost

subMenuItem lines → recurse and multiply by qty

factor for wastagePct and yieldQty if provided

All list endpoints filter by orgId and isActive.
