1) Labour overlay service

New: lib/labour-overlay.ts

getLabourOverlay({ orgId, venueId, weekStart })

For each day/hour in [weekStart .. +6]:

Forecast: hourly revenue/orders (reuse your forecast/day-profile logic).

Shifts: sum scheduled cost per hour (use wage_rate_cents_snapshot || staff.rate).

Rules: target labour % (from labour_rules or org/venue settings).

Return grid:

{ rows: Array<{ date: string, hour: number,
   scheduledCostCents: number,
   forecastRevenueCents: number,
   labourPct: number | null,
   targetPct: number | null,
   headcountByRole: Record<string, number>,
   deficitByRole: Record<string, number> }> ,
  summary: { weekForecastCents: number, weekScheduledCents: number } }


API: GET /api/labour/overlay?weekStart=YYYY-MM-DD

requireOrg({ role: 'manager' }), enforce active venueId.

2) Publish / Unpublish

API: POST /api/roster/publish Body { weekStart }

Transaction:

Find all shifts in week with status='DRAFT' → set status='PUBLISHED', published_at=now(), snapshot wage rate.

Insert roster_publications (increment version if same week republished).

Email notifications (Resend if configured; otherwise [DEV EMAIL] log).

API: POST /api/roster/unpublish Body { weekStart }

Only allow if shifts are in the future. Revert status='DRAFT', delete latest roster_publications for that week.

3) Templates

API:

POST /api/roster/templates Body { name, weekday?, sourceDate }

Read shifts for sourceDate; convert to template lines (minutes from midnight; aggregate by overlapping windows/headcount).

GET /api/roster/templates?weekday?

POST /api/roster/templates/apply Body { templateId, targetDate }

Create DRAFT role shifts on targetDate from lines; staffId null; respect open/close hours.

All endpoints requireOrg({ role:'manager' }) + active venue.

4) Payroll CSV

API: GET /api/workforce/payroll/export?start=YYYY-MM-DD&end=YYYY-MM-DD
CSV columns: staffName, staffExternalId?, date, role, start, end, hours, wageRateCents, costCents

Use wage_rate_cents_snapshot if present; otherwise current rate.
