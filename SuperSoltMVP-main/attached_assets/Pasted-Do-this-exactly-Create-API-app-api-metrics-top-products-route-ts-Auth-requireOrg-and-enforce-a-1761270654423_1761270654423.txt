Do this exactly:

Create API app/api/metrics/top-products/route.ts

Auth: requireOrg() and enforce active venueId (cookie).

Query params:

start (YYYY-MM-DD)

period = day|week|month (defaults to week)

limit (default 10)

Compute [start, end] from period using the same helpers as /api/metrics/kpis.

Compute previous period of equal length for deltas.

Aggregate sales by product for both windows. Use whatever your sales fact table is (you’ve used it in forecasting/KPIs already). Typical shape:

-- adapt names to your schema: daily_sales or order_lines_by_item
SELECT
  ds.menu_item_id,
  mi.name,
  SUM(ds.revenue_cents) AS value_cents,
  SUM(ds.qty) AS qty
FROM daily_sales ds
JOIN menu_items mi ON mi.id = ds.menu_item_id
WHERE ds.org_id = $orgId
  AND ds.venue_id = $venueId
  AND ds.date >= $start AND ds.date <= $end
GROUP BY ds.menu_item_id, mi.name
ORDER BY value_cents DESC
LIMIT $limit;


Do the same for the previous window and compute:

% of Sales = value_cents / total_cents_window * 100

Change = (curr - prev) / NULLIF(prev,0) (return null when prev=0)

Return JSON:

{
  total_cents: number,
  items: Array<{
    id: string,
    name: string,
    value_cents: number,
    pct_of_sales: number,   // 0..100
    change_ratio: number|null // e.g. 0.156 for +15.6%
  }>
}

Replace the mock/top-products data source with the query result.

Loading → same skeleton style as KPIs.
