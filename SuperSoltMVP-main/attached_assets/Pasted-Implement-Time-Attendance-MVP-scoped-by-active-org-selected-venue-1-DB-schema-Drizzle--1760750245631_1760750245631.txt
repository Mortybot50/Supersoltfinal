Implement Time & Attendance (MVP) scoped by active org + selected venue.

1) DB schema (Drizzle)
- Timesheet (
  id uuid pk,
  orgId FK -> Organisation,
  venueId FK -> Venue,
  staffId FK -> Staff,
  shiftId FK -> Shift null,
  clockInTs timestamptz not null,
  clockOutTs timestamptz null,
  breakMinutes int default 0,
  source text CHECK(source IN ('mobile','pin')),
  status text CHECK(status IN ('pending','approved','rejected')) default 'pending',
  managerNote text null,
  createdAt timestamptz default now()
)
Indexes: Timesheet(orgId, venueId), Timesheet(staffId), Timesheet(shiftId)

2) API routes (App Router)
- POST /api/attendance/clock-in
  Body: { staffId, venueId, lat?, lng?, pin? }
  • requireOrg(cookie orgId); verify Staff belongs to same org and venueId matches
  • create Timesheet{ clockInTs=now(), source='mobile' if lat/lng present else 'pin' }
  • withAudit('timesheet.clock_in', null, row)
  • return { timesheetId }

- POST /api/attendance/clock-out
  Body: { timesheetId }
  • requireRole(orgId, ['owner','manager','supervisor']) OR allow same staffId to clock themselves out
  • set clockOutTs=now()
  • withAudit('timesheet.clock_out', before, after)
  • return updated row

- POST /api/attendance/break
  Body: { timesheetId, action: 'start' | 'end', minutes?: number }
  • for 'start' just mark via audit; for 'end' add minutes to breakMinutes (default to minutes or 0)
  • withAudit('timesheet.break_'+action, before, after)
  • return updated row

- GET /api/labour/timesheets?weekStart=YYYY-MM-DD
  • requireOrg(cookie orgId) + venueId (from localStorage header or ?venueId)
  • return latest 200 rows for that week with staff name + status

- POST /api/labour/timesheets/approve
  Body: { timesheetId, approved: boolean, managerNote? }
  • requireRole(orgId, ['owner','manager'])
  • update status to 'approved' or 'rejected', store managerNote
  • withAudit('timesheet.approval', before, after)
  • return updated row

3) UI (very simple)
- New page app/labour/timesheets/page.tsx
  • Table: Staff, Clock in, Clock out, Break (min), Status, Actions
  • Toolbar: week picker (same as roster), approve/reject buttons per row
  • No geo-fence UI yet—this is manager view + basic actions only.
